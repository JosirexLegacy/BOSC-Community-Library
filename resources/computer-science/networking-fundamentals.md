\# Networking Fundamentals — How Computers Talk to Each Other



You use a network every time you open a browser, send a message, or load a file

from the cloud. Understanding how that works underneath makes you a better developer

and a stronger problem-solver.



\---



\## Things Worth Understanding Early



\*\*1. The OSI Model\*\*

Seven layers that describe how data moves from one machine to another.

You don't need to memorize all seven immediately — focus on layers 3 (Network),

4 (Transport), and 7 (Application) first.



\*\*2. IP Addresses and DNS\*\*

Every device on a network has an IP address. DNS translates human-readable names

(like github.com) into those addresses. This is why when DNS fails, nothing works.



\*\*3. TCP vs UDP\*\*

TCP guarantees delivery. UDP is faster but doesn't check.

Video calls use UDP. File downloads use TCP. Knowing why matters.



\*\*4. HTTP and HTTPS\*\*

The protocol your browser uses. HTTPS is encrypted HTTP.

Every web developer needs to understand request/response cycles.



\*\*5. Ports and Sockets\*\*

Ports let one machine run many network services at once.

Port 80 = HTTP. Port 443 = HTTPS. Port 22 = SSH.

\## Free Resources



\- \*\*Computer Networking: A Top-Down Approach\*\* (ask your library)

&#x20; The standard university textbook. Thorough and worth reading.



\- \*\*Cisco Networking Academy (Free)\*\* — https://www.netacad.com/

&#x20; Structured courses with certificates. Works on slow connections.



\- \*\*Professor Messer (YouTube)\*\* — https://www.youtube.com/c/professormesser

&#x20; Clear explanations of networking concepts. Great for CompTIA prep.



\- \*\*Wireshark\*\* — https://www.wireshark.org/

&#x20; Free tool. Capture real network traffic on your machine and \*see\* packets.
